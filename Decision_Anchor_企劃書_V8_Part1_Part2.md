# 決策錨點（Decision Anchor）- 企劃書 V8
## Part 1 決策驗證層 + Part 2 持倉評估層

---

## 一句話定義

**A conditional return distribution engine for evaluating the statistical outcome of a trade under similar historical market conditions.**

條件報酬分布引擎：在給定交易條件下，估計歷史上相似情況的報酬分布，幫助投資者進行統計驗證並評估風險。

---

## 核心原則

- 不進行價格預測
- 不提供投資建議
- 僅基於歷史統計數據進行條件期望評估

```
「在過去相似的市場狀態下，
該交易決策的統計結果如何？」
```

# PART 1：決策驗證層（Pre-Trade Validation Layer）

## 1.1 系統目標

**目的：**

為交易者在決策階段提供條件期望值（Conditional Expectancy）的統計評估，使其能夠在完整信息基礎上進行決策。

**應用場景：**

- 交易者計劃進場前，評估該進場條件的歷史表現
- 識別市場狀態與預期報酬之間的關係
- 量化進場信號的風險回報特徵

---

## 1.2 系統架構

### 狀態編碼（State Encoding）

將市場狀態轉換為離散狀態空間：

```
S_t = (pos_t, mom_t, trend_t)

pos_t：相對位置（Relative Position）
  ├─ High：Close 在 High_20 上方 75% 以上
  ├─ Mid：在中間 25-75% 範圍
  └─ Low：Close 在 Low_20 下方 25% 以內

mom_t：短期動能（Momentum）
  ├─ Strong：Return_{t-5→t} ≥ +5%
  ├─ Neutral：-5% < Return_{t-5→t} < +5%
  └─ Weak：Return_{t-5→t} ≤ -5%

trend_t：中期結構（Trend）
  ├─ Bull：Close > SMA_50
  └─ Bear：Close < SMA_50
```

**狀態空間規模：3 × 3 × 2 = 18 種狀態**

### 歷史相似性匹配（Historical Similarity Matching）

```
Ω(S_t) = {t' | S_{t'} = S_t, t' ∈ historical_period}

該函數識別所有歷史上與當前狀態相同的時刻集合
```

### 條件報酬分布估計（Conditional Return Distribution）

對於相似狀態的所有歷史事件，計算持有期間 H 的報酬分布：

```
r_i(H) = (P_{t_i+H} - P_{t_i}) / P_{t_i}

統計指標：
- P25（下四分位）：風險側報酬
- P50（中位數）：典型報酬
- P75（上四分位）：機會側報酬
- N（樣本量）：統計信度指標
```

### 穩定性分析（Regime Stability Analysis）

沿時間軸分段評估報酬分布的一致性：

```
按年份/期間分組：
  R_y(S_t, H) = {報酬 | 狀態 S_t 在 y 年出現}

輸出分類：
  Stable：跨期間報酬分布一致
  Regime-Dependent：某些市場環境偏好
  Unstable：報酬高度時變
```

## 1.3 決策驗證層的運作流程

### 輸入規格（Input Schema）

```json
{
  "asset_code": "2330",
  "entry_datetime": "2026-03-26T09:30:00Z",
  "technical_state": {
    "relative_position": "High",
    "momentum": "Strong",
    "trend": "Bull"
  },
  "holding_horizon_days": 10
}
```

### 處理步驟

**Step 1：狀態編碼驗證**
- 將用戶輸入的技術指標映射到離散狀態空間
- 確認狀態編碼無歧義

**Step 2：歷史事件檢索**
- 查詢歷史資料庫中狀態相同的所有時刻
- 統計樣本量 N
- 記錄事件發生的年份/期間

**Step 3：報酬分布計算**
- 對每個歷史事件，計算持有 H 天後的報酬
- 計算四分位數、中位數等統計量

**Step 4：穩定性評估**
- 按時間段分組
- 檢驗報酬分布跨期一致性
- 分類為 Stable / Regime-Dependent / Unstable

---

## 1.4 輸出規格（Output Schema）

### 統計指標層（Quantitative Layer）

```
中位數報酬 (P50):     +2.3%
上四分位報酬 (P75):   +11.8%
下四分位報酬 (P25):   -7.5%
樣本量 (N):           28
資料涵蓋期間:         2019-2025
穩定性分類:           Regime-Dependent
```

### 解釋層（Interpretation Layer）

系統根據統計結果自動生成以下評估：

**1. 邊界評估（Edge Assessment）**

根據 P50 與風險特徵進行分類：

```
- 該條件下呈現統計優勢：P50 > 0 且 |P25| < |P75|
- 該條件下未觀察到顯著優勢：P50 ≈ 0 或樣本量不足
- 該條件下呈現統計劣勢：P50 < 0
```

**2. 穩定性評估（Stability Assessment）**

基於穩定性分類：

```
Stable：
「該條件的報酬特徵在不同市場環境中保持一致」

Regime-Dependent：
「該條件的表現與市場狀態高度相關
 當前市場環境對該條件的適配度需要評估」

Unstable：
「該條件的歷史表現不穩定
 過去的統計特徵在未來可能不可靠」
```

**3. 樣本量評級（Sample Robustness）**

```
N ≥ 100：高統計信度
30 ≤ N < 100：中等統計信度
N < 30：低統計信度，稀有狀態
```

## 1.5 使用案例與輸出範例

**案例：交易者評估台積電進場決策**

輸入：
```
股票：2330 (台積電)
進場時間：2026-03-26
技術狀態：
  - 相對位置：High（接近 52 週高點）
  - 動能：Strong（5 日報酬 +6%）
  - 趨勢：Bull（Close > SMA_50）
持有期間：10 天
```

輸出：
```
中位數報酬 (P50):     +2.3%
上四分位報酬 (P75):   +11.8%
下四分位報酬 (P25):   -7.5%
樣本量:               28 次
統計信度:             中等

穩定性分析:
  2019-2020（牛市）：15 次，平均 +5.2%
  2021-2022（熊市）：8 次，平均 -2.1%
  2023-2025（復甦）：5 次，平均 +3.8%
  
  分類：Regime-Dependent
  結論：該條件的優勢主要表現在牛市環境
```

**交易者決策標準**

交易者可基於上述統計結果進行決策，考慮因素包括：
- 預期報酬 (P50) 是否符合目標
- 可承受的風險等級 (P25) 是否在可控範圍
- 當前市場環境與歷史表現的適配度
- 樣本量的統計信度

# PART 2：持倉評估層（Post-Entry Position Assessment Layer / 付費層）

## 2.1 系統目標

**目的：**

為已持倉投資者提供條件概率評估（Conditional Probability Assessment），評估當前持倉在相似歷史場景中的可能路徑。

**應用場景：**

- 交易者評估中途虧損時的持倉管理決策
- 量化未平倉部位的歷史相似性
- 預測未來K天內可能的走向分布

---

## 2.2 持倉評估層的運作流程

### 輸入規格（Input Schema）

```json
{
  "asset_code": "2330",
  "entry_datetime": "2026-03-10T10:15:00Z",
  "entry_price": 550.00,
  "current_datetime": "2026-03-26T14:30:00Z",
  "current_price": 506.00,
  "position_type": "LONG",
  "quantity": 1000
}
```

計算量化指標：
```
Unrealized_PnL = (506 - 550) / 550 = -8%
Days_Held = 16 天
Entry_State = (High, Strong, Bull)  [回溯進場時的狀態]
```

### 處理步驟

**Step 1：持倉狀態復原**
- 根據歷史K線，復原進場時的市場狀態
- 識別該狀態在狀態空間中的分類

**Step 2：條件事件集合檢索**
- 檢索所有相同進場狀態 (High, Strong, Bull) 的歷史事件
- 篩選出「進場後第 n 天出現虧損且幅度相近」的案例
- 樣本集合：Ω_loss = {相似進場條件 + 相似虧損幅度的歷史事件}

**Step 3：多路徑結果分析**
- 追蹤這些歷史事件的後續路徑
- 分類為：反轉、持續虧損、止損等
- 計算各路徑的概率與統計特徵

**Step 4：概率與時間分布**
- 計算各結果的發生概率
- 估計反轉/止損的平均耗時
- 量化最大可能虧損幅度

---

## 2.3 輸出規格（Output Schema）

### 統計指標層（Quantitative Layer）

**當前持倉的條件概率分布：**

```
假設當前：進場 (High, Strong, Bull) 狀態，已虧損 8%，持有 16 天

類似歷史案例統計：
  樣本總數 (N_similar):        12 次
  反轉至正報酬 (Reversal):      7 次 (58.3%)
  持續虧損 (Continued Loss):    3 次 (25.0%)
  止損執行 (Stop-Loss Hit):     2 次 (16.7%)

反轉路徑統計（N=7）：
  平均反轉時間:         9 天
  平均最終報酬:         +4.2%
  中位數反轉報酬:       +3.5%
  最大反轉報酬:         +12.0%

持續虧損路徑統計（N=3）：
  平均最終虧損:         -15.0%
  平均虧損持續時間:     30 天
  最大虧損幅度:         -23.0%

止損路徑統計（N=2）：
  止損執行點位:         -8.5% 至 -9.2%
  事後股價走勢:         -20.0% (平均)
  止損有效性評分:       低
```

### 解釋層（Interpretation Layer）

**1. 持倉路徑評估（Position Path Assessment）**

基於條件概率：

```
高概率路徑（>50%）：
「根據歷史相似案例，該持倉最可能在 9 天內反轉至正報酬
 平均反轉幅度 +3.5% 至 +4.2%」

中概率路徑（25-50%）：
「持續虧損的概率為 25%，平均虧損幅度 -15%，耗時 30 天」

低概率路徑（<25%）：
「止損執行後股價繼續下跌的概率較低，但曾在 2次 歷史案例中發生」
```

**2. 時間維度評估（Temporal Assessment）**

```
T+9天：
  期望值 (Expected Value): 
    0.583 × 4.2% + 0.25 × (-8%)  + 0.167 × (-9%) = +0.47%
    
  最可能結果：反轉至 +3.5% (58% 概率)

T+16天（當前持有期限）：
  期望值：約 +0.8%
  風險評估：下行 25% 概率虧損擴大至 -15%

T+30天（限制持有期限）：
  累積虧損風險：若持續虧損，平均虧損 -15%
```

**3. 退出決策輔助（Exit Decision Support）**

基於持倉評估，系統提供以下決策支持：

```
持有決策：
- 反轉概率 58%，平均反轉時間 9 天
- 建議觀察至 T+9 天以上

風險控制決策：
- 若虧損擴大至 -12% 以上，止損有效性降低
- 歷史案例顯示止損後股價繼續下跌

加碼決策：
- 平均反轉時間 9 天，正期望值為正
- 但需考慮資金容量與風險承受度
```

# PART 4：完整系統框架總結

## 4.1 決策錨點的三層價值

```
層級一：數據層（Quantitative）
  提供 P25, P50, P75 等統計指標
  以及樣本量、穩定性分類
  
  目的：客觀、量化、可驗證

層級二：解釋層（Interpretation）
  自動生成邊界評估、穩定性評估
  為非專業用戶提供易讀的分析
  
  目的：可讀、易理解、可操作

層級三：決策層（Decision Support）
  提供多路徑概率評估
  支持用戶在完整信息基礎上決策
  
  目的：理性決策、風險管理
```

## 4.2 核心優勢與市場定位

**vs 股市老師/顧問：**
```
老師：「買點來了，目標 +30%」
系統：「歷史中位數 +2.3%，下行 -7.5%」
優勢：基於數據，無利益衝突
```

**vs 技術分析軟體：**
```
軟體：「黃金交叉 = 買入信號」
系統：「黃金交叉後的勝率是 45%，中位數 +1.8%」
優勢：量化信號品質，不盲目信號跟隨
```

**vs AI 預測模型：**
```
AI：「預測明天漲到 560」
系統：「相似歷史環境下，10天內報酬分布如下」
優勢：不做不可能的預測，只評估統計規律
```

# 結論

## 決策錨點的本質

```
不是：預測機、交易信號、財務顧問
而是：決策驗證層、風險鏡鑑、統計輔助工具

用戶價值主張：
「在交易決策的關鍵時刻
提供完整、客觀、基於歷史統計的評估
幫助投資者在信息對稱的基礎上做出決策」

市場價值主張：
「打破信息不對稱
提升散戶決策品質
對抗無根據的投資推薦」
```

1. **統計嚴謹性**
   - 175萬矩陣搜索驗證的邏輯

   - 兩層輸出（數據 + 解釋）
   - 可操作的決策支持

3. **法律安全**
   - 不做預測，只陳述歷史
   - 完整的風險聲明

4. **商業可持續**
