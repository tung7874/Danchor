#### 修復內容（`backend/src/data/fetcher.py`）

1. **SSL bypass**：
   - 新增 `import urllib3` + `urllib3.disable_warnings()`
   - `_TWSE_SESSION.verify = False`
   - 原因：TWSE 憑證缺少 Subject Key Identifier，在部分伺服器上 SSL 驗證失敗

2. **TWSE retry**：
   - `fetch_month()` 加 3 次重試 + 指數退避（2s / 4s）
   - timeout 10s → 15s（Render Singapore → Taiwan 延遲）

3. **Worker 限制**：
   - `max_workers` 8 → 4，避免 TWSE rate limit

4. **TPEx 改用 POST**：
   - GET 端點已 302 重導向，改用 POST
   - 支援新格式 `tables[0].data` 和舊格式 `aaData`（向後相容）
   - 注意：TPEx POST API 目前 stock code 不被接受（永遠空），仍會 fallback 到 yfinance

5. **User-Agent 增強**：
   - 完整 Chrome UA + Referer header，避免被 bot 過濾

#### 結論

- **2330.TW（上市）**：TWSE API 正常 → 資料抓取完整 → 分析應正常工作
